Week 2 – SQL Data Analysis & Database Management

Task 1: SQL Analysis on Sample Superstore Dataset

Conducted an end-to-end SQL analysis of the Sample Superstore dataset (approximately 9,994 records) using MySQL Workbench to extract business insights through data filtering, aggregation, and analytical queries.

Key Activities:

Examined the dataset structure and verified data integrity.
Filtered records based on region, category, sales values, and order dates using WHERE conditions.
Performed aggregations with GROUP BY and SQL aggregate functions.
Identified top-performing products, categories, and customers using sorting and ranking techniques.
Detected duplicate records and validated dataset quality through row count and consistency checks.
Generated business-focused insights from sales and customer data.
Task 2: E-Commerce Sales Database Design and SQL Operations

Designed and implemented a relational E-Commerce Sales Database (ecommerce_db) in MySQL Workbench, demonstrating core database design principles and advanced SQL operations.

Key Activities:

Created the database schema and related tables with appropriate relationships.
Implemented Primary Keys, Foreign Keys, and integrity constraints.
Populated the database with sample records for testing and analysis.
Retrieved and analyzed data using SELECT queries with conditional filtering.
Utilized aggregate functions (COUNT, SUM, AVG, MIN, MAX) for statistical analysis.
Applied GROUP BY and HAVING clauses to summarize and filter grouped results.
Executed multi-table JOIN operations to combine related data.
Demonstrated indexing concepts to improve query performance.
Used CASE expressions for conditional data classification.
Illustrated database transaction management using COMMIT and ROLLBACK, along with an understanding of ACID properties.

Platform Used: MySQL Workbench
